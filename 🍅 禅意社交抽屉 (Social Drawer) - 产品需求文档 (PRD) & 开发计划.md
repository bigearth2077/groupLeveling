# 🍅 禅意社交抽屉 (Social Drawer) - 产品需求文档 (PRD) & 开发计划

**项目名称：** 番茄钟自习室 - 社交与通讯录模块 **版本：** v1.0 **状态：** 待开发 **优先级：** High (提升用户留存与互动)

## 第一部分：产品需求说明书 (PRD)

### 1. 项目背景与目标

- **背景：** 用户需要一个不打扰主屏专注，但能随时呼出的社交入口，用于寻找学伴。
- **目标：** 开发一个**侧滑式悬浮抽屉**，集成好友管理、消息处理及核心的“一键约自习”功能。
- **入口：** 点击 Dock 导航栏的“双人/通讯录”图标。

### 2. 功能模块详情

#### 2.1 抽屉容器 (Container)

- **交互：**
  - **滑出：** 点击 Dock 图标，抽屉从屏幕**右侧**平滑滑出。
  - **关闭：** 点击遮罩层 (Overlay)、关闭按钮 (×) 或再次点击 Dock 图标。
- **视觉：** 高斯模糊 (Glassmorphism) 背景，与个人资料卡片风格统一。

#### 2.2 快捷入口区 (Quick Links)

位于抽屉顶部，作为功能分流：

1. **自习大厅 (Lobby)：** 点击跳转至公共自习室列表页。
2. **排行榜 (Ranking)：** 点击跳转至全站/好友排行榜页面。

#### 2.3 搜索与添加 (Search & Add)

- **搜索框：** 输入用户 ID 或昵称。支持回车搜索。
- **添加按钮 (+)：**
  - 点击后调用 `POST 发送好友请求`。
  - **反馈：** 发送成功后显示 Toast 提示，按钮暂时变灰或变为对钩。

#### 2.4 列表分栏 (Tabs)

- **Tab 1: 好友 (Friends)**
  - 显示好友总数。
  - **默认选中**项。
- **Tab 2: 请求 (Requests)**
  - 显示待处理请求数量。
  - **红点逻辑：** 当 `待处理请求数 > 0` 时，Tab 上显示红色数字角标 (Badge)。

#### 2.5 好友列表 (核心交互)

- **展示项：** 头像、昵称、在线状态、当前状态描述（如“专注中 25:00”）。
- **状态排序逻辑：** 专注中 > 在线 > 离线。
- **右键菜单 (Context Menu) —— 关键功能：**
  - 用户在好友项上**点击右键**，触发自定义菜单（屏蔽浏览器默认菜单）。
  - **菜单项：**
    1. 🍅 **一键邀请共学：** 核心功能。调用创建房间接口并发送邀请。
    2. 👤 **查看资料：** 弹窗展示该用户的公开信息。
    3. 💔 **解除好友：** 二次确认后删除。

#### 2.6 请求列表 (Request List)

- **展示项：** 头像、昵称、请求附言。
- **操作：**
  - **同意：** 调用处理接口 (Action: Accept)，成功后该条目移至好友列表。
  - **忽略：** 调用处理接口 (Action: Reject)，条目消失。

## 第二部分：API 映射表 (基于接口清单)

| 模块     | 动作         | 对应 API 接口                                 | 触发时机               |
| -------- | ------------ | --------------------------------------------- | ---------------------- |
| **搜索** | 查找用户     | `GET /friend/search` (搜索用户)               | 点击搜索或回车         |
| **添加** | 发送请求     | `POST /friend/request/send` (发送好友请求)    | 点击 + 号按钮          |
| **列表** | 获取好友     | `GET /friend/list` (我的好友列表)             | 打开抽屉 / 切换 Tab    |
| **列表** | 获取请求     | `GET /friend/request/pending` (待处理请求)    | 打开抽屉 / 切换 Tab    |
| **请求** | 处理请求     | `POST /friend/request/handle` (处理好友请求)  | 点击同意/拒绝          |
| **右键** | 删好友       | `DEL /friend/delete` (解除好友)               | 右键菜单确认删除       |
| **右键** | **一键拉房** | `POST /room/create` (创建房间) + 邀请逻辑     | **右键菜单“一键邀请”** |
| **右键** | 看资料       | `GET /user/profile/public` (查看他人公开资料) | 右键菜单“查看资料”     |

## 第三部分：开发实施计划 (Roadmap)

### Phase 1: 基础框架构建 (UI)

**目标：** 实现抽屉的开关动画与静态布局。

1. **组件开发：** 封装 `SocialDrawer` 组件。
2. **CSS 动画：** 实现 `transform: translateX` 的平滑过渡，确保性能（使用 GPU 加速）。
3. **右键菜单组件：** 开发一个通用的 `ContextMenu` 组件，支持定位计算（防止菜单超出屏幕边界）。

### Phase 2: 核心数据对接

**目标：** 让列表显示真实数据。

1. **数据获取：** 组件挂载时并发请求 `好友列表` 和 `待处理请求`。
2. **状态渲染：** 根据 API 返回的 `status` 字段渲染不同颜色的状态点（绿/红/灰）。
3. **搜索逻辑：** 实现简单的防抖搜索。

### Phase 3: 交互逻辑深化

**目标：** 实现社交互动的闭环。

1. **右键邀请逻辑：**
   - 前端逻辑：点击“邀请” -> Loading -> 成功提示“已发送邀请，等待对方加入...”。
   - *注：需确认后端是否支持直接创建并邀请，否则需分两步走。*
2. **请求处理：** 实现“同意”后的列表乐观更新（Optimistic Update），即点击同意后立即在界面上把人挪到好友列表，无需等待接口返回。

### Phase 4: 实时性优化 (进阶)

**目标：** 状态同步。

1. **轮询或 Socket：** 每隔一定时间（如 30s）或通过 WebSocket 更新好友的在线/专注状态。
2. **红点同步：** 即使抽屉关闭，Dock 栏上的图标也应显示请求数的红点。

## 第四部分：注意事项 (Notes)

1. **右键菜单体验：** 在移动端（如有适配计划）不支持右键，需考虑长按（Long Press）或在列表项右侧增加“...”更多操作按钮作为替代方案。
2. **空状态 (Empty State)：** 当没有好友或没有请求时，需展示友好的插画提示（如：“还没有好友？去自习大厅看看吧”）。
3. **专注状态干扰：** 如果好友状态为“专注中（红色）”，建议在右键“邀请”时弹出提示：“对方正在专注中，确定要打扰吗？” 这是一个非常贴心的细节。
