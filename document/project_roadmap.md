## 🏗️ 代码质量改进建议

### 后端
1. **错误处理统一**：目前部分 Handler 返回 400、部分返回 500，建议定义统一的业务错误码体系
2. **Service 层解耦**：部分 Service 直接实例化（`&service.AuthService{}`），建议引入依赖注入
3. **请求校验增强**：为所有 DTO 添加完整的 binding tag 校验
4. **日志规范**：统一使用结构化日志，替代 `log.Printf`

### 前端
1. **API 层统一**：`feature/*/api.js` 的导出风格不一致，建议统一返回格式
2. **状态管理**：部分页面使用本地 state 管理复杂状态（如 Home.jsx），建议迁移到 Zustand
3. **组件拆分**：Profile.jsx（640行）过于庞大，建议按 Tab 拆分为子组件
4. **TypeScript 迁移**：当前使用 JSX，建议逐步迁移到 TSX 增强类型安全
